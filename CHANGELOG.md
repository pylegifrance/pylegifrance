## [1.4.0] - 2026-04-18

### Features

- **Contributing workflow switched to PR-first** (issues are disabled on the
  repo). New `.github/PULL_REQUEST_TEMPLATE.md` with a canonical "Prompt LLM
  utilisé" section for transparency when a contribution was drafted with an
  LLM (#50).
- **LLM prompt disclosure** archived under `docs/raw/prompts/` as source
  material for the LLM-maintained wiki (#48, #50).

### Bug Fixes

- `.github/CONTRIBUTING.md` no longer recommends `gh pr create --fill` (which
  bypasses the PR template); replaced by plain `gh pr create` with an
  explicit warning (#50).

### Refactoring

- **Documentation site migrated from MkDocs Material to Astro Starlight** as
  an LLM-maintainable wiki. Flat structure keyed on library concepts
  (entities / concepts / operations / references) rather than the former
  Diátaxis quadrants. French default locale, English mirror under `/en/`
  with automatic fallback. `starlight-llms-txt` plugin emits `/llms.txt` and
  `/llms-full.txt` for AI-agent consumption (#48).

### Documentation

- `.claude/rules/*` refreshed to reflect the new toolchain (Starlight,
  release-please, `conventional-pre-commit`); every rule now cites its
  official-docs source. Cross-repo file references switched to Claude
  Code's `@path` syntax (#48).
- New concept page `cid-and-versioning` documenting the CID pattern,
  consolidation principle, and full legal-status enum, sourced from the
  official DILA `lexique-api-lgf.docx` (#48).
- New concept page `api-terms-and-quotas` documenting the 95% prod SLA,
  PISTE quotas, Etalab 2.0 data licence, credential-compromise procedure,
  and CNIL/ANSSI security baseline, sourced from the 2022 v1.0 DILA Terms
  of Use (#48).
- `docs/raw/legifrance/` archives the five official DILA reference
  documents (lexique, exemples, tris/filtres, 2022 CGU PDF + docx, 2020
  PISTE CGU PDF) with `.txt` companions for LLM ingest (#48).
- `entities/article` and `references/juri` corrected against the actual
  code; `entities/fond-code` and others regenerated from the real Pydantic
  model instead of the inherited dict-shape from the old MkDocs docs (#48).

### CI / Release automation

- **`release-please`** is wired up (`.github/release-please-config.json`,
  `.github/release-please-manifest.json`) to parse Conventional Commits and
  maintain a rolling release PR. From 1.4.1 onwards, version bumps and
  CHANGELOG updates are automated (#48).
- **Single `test` job** on Python 3.12 (the floor declared in
  `pyproject.toml`). Matrix dropped (#49).
- **Deduplicated CI triggers**: `test.yml` now fires once per change
  (PR to `main` or push to `main`), not twice (#50 series).
- **Docs deploy workflow** (`docs.yml`) publishes the Starlight site to
  GitHub Pages on every push to `main` touching `docs/**` (#48).
- **Commit-message linting** via `conventional-pre-commit` at the
  `commit-msg` stage, aligned with the release-please type set (#48).

### Project layout

- Release-please config moved from repo root to `.github/` alongside the
  workflow that consumes it.
- GitHub Actions all bumped to latest majors:
  `actions/checkout@v6`, `actions/deploy-pages@v5`,
  `withastro/action@v6`, `googleapis/release-please-action@v4`.
